# TokenBar — Krill Login: Fix repeated popup + keyboard focus

## Goal
Fix: (1) Login window pops up every time menu opens (2) Still can't type in input fields.

## Root Cause Analysis

### Bug 1: Repeated auto-popup
`KrillUsageFetcher.fetchUsage()` (line 13-16) calls `loginViaWebView()` when there's no stored JWT. Every fetch = login window. There's already an explicit login flow via `KrillProviderImplementation.runLoginFlow()` — the auto-popup on fetch is redundant and aggressive.

**Fix**: In `KrillUsageFetcher.fetchUsage()`, if no stored JWT, return `buildEmptySnapshot()` immediately. Do NOT call `loginViaWebView()`. The user triggers login explicitly from the menu "Login" button.

### Bug 2: Keyboard focus not routing to WebView
The `Task { @MainActor }` block in `setupWindow()` runs asynchronously — by the time it executes, the runloop may have already processed other events and the activation window is missed. Need synchronous activation.

**Fix**: Replace the async Task block with synchronous calls in `setupWindow()` (which is already @MainActor):
```swift
NSApp.activate(ignoringOtherApps: true)
RunLoop.current.run(until: Date(timeIntervalSinceNow: 0.15))
window.orderFrontRegardless()
window.makeKey()
window.makeFirstResponder(webView)
```
This also eliminates the fire-and-forget Task bug (no more repeated popup from activation race).
